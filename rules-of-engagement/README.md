# Rules of Engagement — Automated Stale Account Workflow

Automates the monthly NORAM stale-account review cycle at Checkout.com:

1. **Salesforce** queries accounts where `Stale_Account__c = 1` and POSTs a Slack Block Kit digest to the NORAM sales pod group DM via Incoming Webhook.
2. **Sales pod leaders** (+ BPs) click **DROP** or **KEEP** on each account in Slack.
3. **AWS Lambda** receives the button interaction, verifies it, and writes back to Salesforce:
   - **DROP** → sets `Account.OwnerId` to Sales Operations (`0051p00000AtuwrAAB`)
   - **KEEP** → clears `Stale_Account__c` so the account won't appear next month
4. An ephemeral Slack confirmation is sent back to the user who clicked.

## Setup Order

### 1. Slack App

1. Go to [api.slack.com/apps](https://api.slack.com/apps) → **Create New App** → From scratch.
2. Enable **Incoming Webhooks** — create a webhook for the NORAM sales pod group DM. Copy the URL.
3. Enable **Interactivity & Shortcuts** — leave the Request URL blank for now (set after step 3).
4. Install the app to your workspace.

### 2. AWS — Terraform

```bash
# First time: package the Lambda
bash rules-of-engagement/lambda/build.sh

# Deploy infrastructure
cd rules-of-engagement/terraform
terraform init
terraform plan
terraform apply
```

After `apply`, copy the `slack_request_url` output value.

**Populate secrets in SSM** (one-time, out-of-band):

```bash
aws ssm put-parameter \
  --name "/rules-of-engagement/slack/signing_secret" \
  --value "YOUR_SLACK_SIGNING_SECRET" \
  --type SecureString --overwrite

aws ssm put-parameter \
  --name "/rules-of-engagement/salesforce/instance_url" \
  --value "https://checkout.my.salesforce.com" \
  --type SecureString --overwrite

# Repeat for: client_id, client_secret, username, password, security_token
```

### 3. Slack App — Request URL

Paste the `slack_request_url` Terraform output into:  
**Slack App → Interactivity & Shortcuts → Request URL**

Save and verify (Slack will send a test POST; Lambda must return 200).

### 4. Salesforce

Follow `salesforce/flow_setup_guide.md` to:
- Add `https://hooks.slack.com` to Remote Site Settings
- Build the Scheduled Flow that queries stale accounts and POSTs to Slack
- Set up a Connected App so the Lambda can authenticate and write back

## Environment Variables (Lambda)

Set automatically by Terraform. All point to SSM SecureString paths.

| Variable | SSM Path |
|---|---|
| `SLACK_SIGNING_SECRET_PARAM` | `/rules-of-engagement/slack/signing_secret` |
| `SF_INSTANCE_URL_PARAM` | `/rules-of-engagement/salesforce/instance_url` |
| `SF_CLIENT_ID_PARAM` | `/rules-of-engagement/salesforce/client_id` |
| `SF_CLIENT_SECRET_PARAM` | `/rules-of-engagement/salesforce/client_secret` |
| `SF_USERNAME_PARAM` | `/rules-of-engagement/salesforce/username` |
| `SF_PASSWORD_PARAM` | `/rules-of-engagement/salesforce/password` |
| `SF_SECURITY_TOKEN_PARAM` | `/rules-of-engagement/salesforce/security_token` |
